# Orchestrator UI

This folder contains the User Interface (UI) for the project's main orchestrator and its corresponding documentation. 
The UI is designed to simplify the generation of configuration files and the manual execution of the complete Firmware Lens pipeline, offering a centralized control panel.

## 🚀 UI Features

Through the graphical interface, you can:
1. **Select the Project Folder**: The root path of the firmware or software to be analyzed.
2. **Choose the Project Type (Toolchain)**: Select which toolchain manages the project (e.g., `Keil`, `CMake`/`GCC`). This allows the tool to infer how to extract definitions (for instance, automatically looking for a `.uvprojx` file for Keil or a `compile_commands.json` for CMake).
3. **Generate a Coherent Configuration**: Once parameters are selected, clicking the specific button will generate a `config.json` file *inside the selected project folder*.
4. **Output Routing (Docs and Logs)**:
   - **Docs**: All files generated by the scripts (intermediate JSON files, Markdown output) will be configured to be saved inside a `docs/` folder within the project folder.
   - **Logs**: All orchestrator execution logs will be saved inside a `logs/` folder within the project folder.
5. **Run the Pipeline**: directly start the `run_all.py` script using the newly generated configuration file, displaying real-time process progress without opening additional terminals.

## 🛠️ How to run the UI

The UI can be started in two ways:

### 1. Execution via Python Script
If you have Python installed, you can launch the script directly. The UI uses standard libraries (like `tkinter`), so it works out-of-the-box without requiring additional packages (beyond those required by the main project):

```bash
python run_ui.py
```

### 2. Execution via Standalone Executable (.exe)
For greater convenience, we provide a pre-compiled `.exe` file that does not require launching a manual terminal. 
You can find it inside the `dist/` folder:
```bash
dist/OrchestratorUI.exe
```
> **Important Note:** the executable wraps the graphical interface, but to launch the deep pipeline (`run_all.py`), it needs to access the Python source files of the rest of the project. Therefore, the executable must always be kept inside the `dist/` folder or placed in the root folder (`firmware-lens/`). If you want it on your Desktop, we recommend creating a *shortcut* to the executable rather than moving the file itself.

---

## ⚙️ Recompiling the Executable
If you modify the `run_ui.py` script in the future and want to regenerate the updated `OrchestratorUI.exe`, use the PyInstaller tool.
From the current `orchestrator/artifact/` folder, run:
```bash
pip install pyinstaller
pyinstaller --onefile --noconsole --name OrchestratorUI run_ui.py
```

## 📄 Generated Configuration Structure

When you click on "Generate config.json", the UI creates a JSON file with this base structure (adapted to the selected project and localized by its relative path):

```json
{
  "toolchain": "keil",
  "uvprojx": "<PROJECT_FOLDER>/my_project.uvprojx",
  "project_type": "firmware",
  
  "log_dir": "<PROJECT_FOLDER>/logs",
  
  "compile_commands": "<PROJECT_FOLDER>/docs/compile_commands.json",
  "functions_index": "<PROJECT_FOLDER>/docs/functions_index.json",
  "function_categories": "<PROJECT_FOLDER>/docs/function_categories.json",
  "call_graph": "<PROJECT_FOLDER>/docs/call_graph.json",
  "tasks": "<PROJECT_FOLDER>/docs/tasks.json",
  "task_call_graph": "<PROJECT_FOLDER>/docs/task_call_graph.json",
  "firmware_ir": "<PROJECT_FOLDER>/docs/firmware_ir.json",
  "functions_detail_dir": "<PROJECT_FOLDER>/docs/functions_detail",
  "architecture_overview_md": "<PROJECT_FOLDER>/docs/ARCHITECTURE_OVERVIEW.md"
}
```
This way, all the materials produced by Firmware Lens (both intermediate JSON files and Markdown reports) stay neatly contained inside your active project's workspace.
